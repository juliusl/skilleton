### Task 1.3: Define Hierarchy types    [medium]

**Description:** Define the four Hierarchy types in `src/types/hierarchy.rs`: `Task`, `Step`, `Procedure`, and `Skill`. Each struct composes `ItemMeta` for common metadata. Hierarchy relationships use typed `Vec` fields: `Step.tasks: Vec<Task>`, `Procedure.steps: Vec<Step>`, `Skill.procedures: Vec<Procedure>`. Each hierarchy level also has `policies: Vec<Policy>` for Singleton attachment. `Step` has `completion_criteria: Vec<CriterionRef>`. `Procedure` has `entrance_criteria` and `exit_criteria`. `Skill` includes `metadata: SkillMeta`.

**Implementation Notes:**

```rust
// src/types/hierarchy.rs
use super::{ItemMeta, CriterionRef, Policy, SkillMeta};

#[derive(Debug, Clone)]
pub struct Task {
    pub meta: ItemMeta,
    pub subject: String,
    pub action: String,
}

#[derive(Debug, Clone)]
pub struct Step {
    pub meta: ItemMeta,
    pub tasks: Vec<Task>,
    pub completion_criteria: Vec<CriterionRef>,
    pub policies: Vec<Policy>,
}

#[derive(Debug, Clone)]
pub struct Procedure {
    pub meta: ItemMeta,
    pub steps: Vec<Step>,
    pub entrance_criteria: Vec<CriterionRef>,
    pub exit_criteria: Vec<CriterionRef>,
    pub policies: Vec<Policy>,
}

#[derive(Debug, Clone)]
pub struct Skill {
    pub meta: ItemMeta,
    pub metadata: SkillMeta,
    pub procedures: Vec<Procedure>,
    pub policies: Vec<Policy>,
}
```

**Test & Acceptance Criteria:**
- [x] `Task` has fields: `meta`, `subject`, `action`
- [x] `Step` has typed children: `tasks: Vec<Task>`, plus `completion_criteria` and `policies`
- [x] `Procedure` has `steps: Vec<Step>`, `entrance_criteria`, `exit_criteria`, and `policies`
- [x] `Skill` has `procedures: Vec<Procedure>`, `policies: Vec<Policy>`, and `metadata: SkillMeta`
- [x] Compile-time type safety: a `Step` cannot contain a `Procedure` (enforced by `Vec<Task>` type)
- [x] All types derive `Debug` and `Clone`
- [x] `cargo build` succeeds with no warnings

**Dependencies:** Task 1.2

**ADR Reference:** ADR-0002, Decision §1 (concrete structs), §3 (typed Vec hierarchy), §4 (Singleton attachment), §5 (explicit Policy inheritance)
