## Stage 1: ItemId Parsing and Validation

### Task 1.1: Implement ItemId parsing and validation    [medium]

**Description:** Refine the placeholder `ItemId` into a structured hierarchical path type. Implement parsing from strings like `skill:my-skill.procedure:auth.step:validate`. Validate: type prefixes must be one of `skill`, `procedure`, `step`, `task`, `policy`, `criterion`; slugs must be lowercase, hyphenated, max 50 chars per segment. Implement `Display` for serialization back to string form. Add a `segments()` method that returns the parsed segments. Add a `parent()` method that returns the path without the last segment. Add a `prefix_matches()` method for scope resolution.

**Implementation Notes:**

```rust
#[derive(Debug, Clone, PartialEq, Eq, Hash)]
pub struct ItemId {
    raw: String,
}

#[derive(Debug, Clone, PartialEq, Eq)]
pub struct Segment {
    pub type_prefix: TypePrefix,
    pub slug: String,
}

#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum TypePrefix {
    Skill,
    Procedure,
    Step,
    Task,
    Policy,
    Criterion,
}

impl ItemId {
    pub fn parse(s: &str) -> Result<Self, ItemIdError>;
    pub fn segments(&self) -> Vec<Segment>;
    pub fn parent(&self) -> Option<ItemId>;
    pub fn prefix_matches(&self, prefix: &ItemId) -> bool;
    pub fn append(&self, type_prefix: TypePrefix, slug: &str) -> Result<ItemId, ItemIdError>;
}
```

**Test & Acceptance Criteria:**
- [x] `ItemId::parse` accepts valid paths like `skill:my-skill.procedure:auth`
- [x] `ItemId::parse` rejects invalid type prefixes (e.g., `foo:bar`)
- [x] `ItemId::parse` rejects slugs with uppercase, spaces, or >50 chars
- [x] `ItemId::parse` rejects empty strings and malformed segments
- [x] `segments()` returns the correct parsed segments
- [x] `Display` implementation round-trips: `parse(id.to_string()) == id`
- [x] `parent()` returns `None` for single-segment paths
- [x] `prefix_matches()` correctly identifies scope containment
