### Task 1.3: Fix `detect_policy_overlaps` false positive    [medium]

**Description:** `check_same_level_policies` flags complementary policies at the same scope as overlapping when they lack explicit `compatible_with` annotations. In the onboarding fixture, `policy:be-friendly` and `policy:no-jargon` are independent constraints that don't conflict — the false positive forces unnecessary `compatible_with` boilerplate.

**Fix approach:** Change the default assumption from "same-level policies are potentially conflicting" to "same-level policies are compatible unless they share overlapping concerns." Since automated semantic analysis of policy text is out of scope, use a simpler heuristic: only report same-level overlaps when 3+ policies exist at a scope AND they lack pairwise `compatible_with` annotations. Two policies at the same level are common and usually complementary; three or more signals a density that warrants explicit compatibility declarations.

**Alternative considered:** Remove same-level checking entirely. Rejected — it does catch genuine conflicts when policies accumulate.

**Files to modify:**
- `src/conflict.rs` — update `check_same_level_policies` threshold

**Implementation Notes:**
```rust
fn check_same_level_policies(

    scope: &ItemId,
    overlaps: &mut Vec<PolicyOverlap>,
) {
    // Only check when 3+ policies exist at the same level.
    // Two policies at the same level are common and usually complementary.
    if policies.len() > 2 {
        // ... existing pairwise compatibility check
    }
}
```

**Test & Acceptance Criteria:**

### Task 2.2: Create default template and refactor `render_skill()`    [medium]

**Description:** Define a `DEFAULT_TEMPLATE` constant that reproduces the current `render_skill()` output. Refactor `render_skill()` to call `render_skill_with_template(skill, DEFAULT_TEMPLATE)`. The new function builds a `RenderContext` and applies the template via `ramhorns`.

**Files to modify:**
- `src/render.rs` — add `DEFAULT_TEMPLATE`, `render_skill_with_template()`, refactor `render_skill()`

**Implementation Notes:**
The default template must produce byte-for-byte identical output to the current hardcoded rendering for all existing test cases. This requires careful attention to:
- Newline placement (blank lines between sections)
- Blockquote format for policies (`> **id**: text`)
- List format for criteria (`- **id**: description`)
- Conditional section omission
- Conditions annotation format (`*Conditions: ...*`)

```rust
pub const DEFAULT_TEMPLATE: &str = "..."; // Multi-line template string

pub fn render_skill(skill: &Skill) -> String {
    render_skill_with_template(skill, DEFAULT_TEMPLATE)
}

pub fn render_skill_with_template(skill: &Skill, template: &str) -> String {
    let ctx = RenderContext::from(skill);
    let tpl = ramhorns::Template::new(template).expect("default template must parse");
    tpl.render(&ctx)
}
```

**Test & Acceptance Criteria:**
- [x] `render_skill()` signature unchanged — backwards compatible
